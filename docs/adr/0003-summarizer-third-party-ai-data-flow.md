# Summarizer が報告本文を第三者AIプロバイダへ送る際のデータフロー（実行時／経路B）

**ステータス: 提案中（スタブ・法務/契約の確定待ち）**

## 背景

プロダクトの中核機能 `報告→要約` は、Summarizer 抽象化層を通じて報告本文を外部 AI プロバイダへ送信する（実行時＝経路B）。報告本文には第三者PII・要配慮個人情報（L2）が入りうる（§6 データ階層参照）。これは開発時の取り扱い（M-2＝経路A、`docs/` の CONTEXT.md／計画書 §6 で確定済み）とは別問題であり、不可逆かつ法務判断を伴うため独立の ADR とする。開発ルールと同じ節に混ぜると双方が強制不能になるため分離した。

## 決定（未確定・確定待ちの論点）

以下が確定するまで本 ADR は「提案中」に留め、確定後に本文を書き換える。

1. **プロバイダ選定とデータ処理契約（DPA）**：要配慮個人情報の処理を委託できる契約条件か。
2. **保存・学習オプトアウト**：送信データがモデル学習・ログ保持に使われない設定になっているか。
3. **要配慮情報の第三者提供の適法性**：本人同意・利用目的の範囲、越境移転（送信先リージョン）の扱い。
4. **送信前の最小化**：Summarizer 側で不要な L2 フィールドを送らない／マスキングして送る設計余地。
5. **監査証跡**：いつ・どの報告が外部送信されたかのログ要件。

## 影響

- 未確定のまま本番で要配慮領域の実データを要約すると、要配慮情報の不適切な第三者提供リスクがある。→ **本番化フェーズ（移行後）の着手ゲート**とする。
- MVP（TS）〜移行期は合成データのみ（経路A・例外なし）で開発するため、本 ADR の未確定は開発進行をブロックしない。
