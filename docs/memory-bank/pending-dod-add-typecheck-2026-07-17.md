# CLAUDE.md 昇格候補: 完了の定義に typecheck を足す

> **PM 承認待ち。** 憲法（CLAUDE.md）§2 の変更提案。自律エージェントは本体に直接書かない（/flywheel 手順4）。
> 起票日: 2026-07-17 / 起案: 下流セッション（/flywheel）

## 観察

`main` の CI が PR #11（slice-01）のマージから **4回連続で赤**だった（#17）。赤の1つは typecheck で、`prismaReportRepository.ts` が参照する `model Report` が `schema.prisma` に存在しなかった。

この赤は **3回のマージをすり抜けた**。理由は憲法 §2 の完了の定義（4判定）に **typecheck が含まれていない**ため。4判定は「受け入れテスト緑・golden・全ディレクトリに diff・秘密なし」であり、型は誰も見ていない。受け入れテストは `PERSISTENCE=memory` で走るので緑のままだった。

しかも issue #17 は「Prisma 実装は一度も実行されないまま型だけが壊れているコード」と書いていたが、**これは誤り**だった。`apps/service/src/main.ts:16` の永続化の既定は `prisma` で、`PERSISTENCE=memory` は開発・受け入れテスト用の分岐にすぎない。つまり**本番の既定経路が壊れたまま3回マージされていた**。

## なぜ憲法の問題か

§2 は「『動いたように見える』は完了ではない。以下4点が揃って初めて報告する。**すべて機械判定**」と宣言している。しかしその4判定は、**コンパイルが通ることを含んでいない**。受け入れテストが緑でも型が壊れている状態を、憲法は「完了」と呼んでしまう。

これは §2 が戒めている「動いたように見える」の一種である。テストが通る経路（memory）と本番の経路（prisma）が違うとき、テストの緑は本番の健全性を何も保証しない。

## 提案（差分）

CLAUDE.md §2 の4判定に1行足す:

```diff
  1. 受け入れテストが緑（テスト出力の**生ログを提示**する）。**API 層と UI 層の両方**（ADR-0018）
  2. golden スクリーンショットとの差分が閾値内。**撮影不可のスライスは DOM アサーションが緑**（ADR-0008・0018）
  3. **指示書「3. 触ってよいファイル範囲」の全ディレクトリに diff がある。触らずに緑なら停止**（ADR-0018）
- 4. シークレット・PII が出力/差分に無い
+ 4. **typecheck が緑**（`pnpm typecheck`）。テストが通る経路と本番の経路は違う。緑は型の健全性を保証しない
+ 5. シークレット・PII が出力/差分に無い
```

見出しも「以下4点」→「以下5点」に変わる。

## 剪定（1行足すので1行削る・/flywheel 手順5）

§6 の次の行を削る候補として挙げる:

```
- **`.mcp.json` で `${workspaceFolder}` は展開されない**（VS Code 記法）。環境変数展開のみ対応。
```

基準は「この行を消したら Claude はミスをするか？」。`.mcp.json` は既に設定済みで、下流・上流のどのコマンドも `.mcp.json` を編集しない（編集するのは AIアーキが箱を作るときだけ＝キックオフ時の一度きり）。**セットアップ時の落とし穴であり、日々の作業では発火しない。** 消してもミスは起きない。

ただし判断は PM に委ねる。§6 は「非自明な落とし穴」の集積であり、消す基準の適用は起案者より PM のほうが文脈を持つ。

## 補足: なぜ「宣言」で止めるのか（強制力の階段）

typecheck は既に CI にジョブとして存在する（`.github/workflows/ci.yml`）。つまり**事後検証はある**。欠けているのは「下流が報告する前に自分で見る」という宣言だけである。

したがって階段は「なし → **宣言**」の1段でよい。hook 化（実行時強制）は飛び級になる。CI が緑を担保している以上、憲法に書いて下流が自分で見るようにすれば足りる。

なお #17 の本文自身が「憲法 §2 の完了の定義に typecheck が含まれていないことも、この赤が3回のマージをすり抜けた一因。判定に足すかどうかも論点（`/flywheel`）」と書いており、**本 issue が2ストライク目**にあたる（1回目 = #17 の起票、2回目 = 本セッションで本番経路の破損として実証）。
