# プロセスKPI メトリクスログ

目的A（再現可能な型の確立）の成否を測るプロセス指標の記録場所。詳細は計画書 §1「成功基準（プロセスKPI）」。
層境ゲートの判定記録（GO/NO-GO・代理判定）は別ファイル **`gates.md`**（ADR-0007）。
`/submit` がスライス完了時に1行追記する（`救援有無` だけはリーダーが窓口対応時に記録＝初級の自己申告を避ける）。
所有は Harness-Keeper（AIアーキ）。レビューはスプリント末の骨格見直し（計画書 §5・§8）。

## 記録フォーマット（1スライス＝1行）

| slice_id | 日付 | 救援有無(y/n) | 再作成回数 | 枠消費 | 差し戻し理由 | メンバー |
|---|---|---|---|---|---|---|
| slice-01-report-create | 2026-07-xx | n | 0 | — | — | （初回完走者） |

- **救援有無**: ループ途中で人が助けたか（リーダーへの一次質問を含む）。y は AFK未完走。← リーダー記録。
- **再作成回数**: セッションを捨てて `/pickup` から再開した回数。2回超はスライス肥大シグナル。
- **枠消費**: 1スライスの Pro 枠消費（$/task の代理）。自動取得（`/cost` 等）。
- **差し戻し理由**: 統合役再検証NG／層境ゲートNO-GO の理由（あれば）。

## 集計（北極星＝AFK完走率）

- **AFK完走率** = 救援なし(n)で緑＋PR 到達したスライス数 ÷ 全スライス数。**直近5の移動平均**で追う。
- 一次成功条件: 移動平均が右肩上がり。暫定ゲート: MVP 終盤で ≥70%（slice-01〜05 実測後に本較正）。
- 補助指標: 差し戻し率／スライス肥大率（再作成2回超）／枠効率。北極星が割れたときの原因切り分け用。

## C（従・非ゲート）

各下流メンバーの **Time-to-first-green**（初回スライス完走までの日数）を別途記録。初スプリント内に全員1回の完走を目標（未達でも計画は止めず A側の欠陥シグナルとして Flywheel へ）。

## 指標が割れたら

罰でも即作業でもなく Flywheel の観察項目。必ず原因を **「スライス設計の欠陥（大きすぎ・受入基準過多）」か「ハーネスの欠陥（枠・skills・hooks）」** に2分類してから対策。CLAUDE.md/hook 昇格は2ストライク→昇格の階段に載せる。
