# slice-18 voice-input-stt — 音声入力・STT（テキスト化 → 本文集約・確定前に修正可能）

> 指示書の正本（ADR-0006）。issue 本文はポインタのみ。仕様表: `docs/spec/slice-18.md`（approved 2026-07-19）。依存: slice-01（`POST`/`PATCH /reports`・S3 業務報告入力）。フェーズ: 5。**S3 業務報告入力の拡張。**
> **PM 決定（2026-07-15）**: STT 方式は **Web Speech API（クライアント側）**。差し替え可能な抽象層の裏に置く（サーバ側 STT は後から足せる形）。
> **重要（frontend 専用）**: クライアント側完結が既定のため、**STT 用の新規サーバ REST は立てない**（本文は既存 `POST`/`PATCH /reports` で保存）。よって本スライスは **api.spec 無し・オラクル変更なし**。工程4 は ui.spec のみ（correctness は工程6 の実 frontend で検証）。

## 1. ゴール

S3 業務報告入力（`/reports/new`）に音声入力を足す。自由文モードで **音声入力の起動操作**があり、既存の本文入力欄と併用できる。STT（Web Speech API・クライアント側）の結果は **本文へ取り込む前に確認・修正できる領域**に提示され、ユーザーが取り込みを操作して初めて **本文末尾へ追記**される（自動確定しない）。録音権限拒否・STT 失敗時は、**入力済み本文を保持**したまま **失敗をテキストで**（色のみに依存せず）提示し、キーボード入力を継続できる（degrade で報告は必ず出せる）。本文の保存は既存 `PATCH /reports`（新規サーバ契約は足さない）。
