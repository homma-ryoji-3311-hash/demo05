# slice-08 skillsheet-generate — スキルシート（Excel）生成

> 振る舞いの正本: `reference-mock/spec.md §3.5`（生成の3フェーズ分離・出力ファイル名・格納と履歴）。
> REST 表現（パス/メソッド/コード）は参照モックに無い＝🟥 `source: PM`（overview §3）。フェーズ: Phase 2。

## grill 決定ログ（2026-07-18・PM）

1. **生成入力＝合成マスター元データ fixture。** slice-08 は[[マスター元データ]]の**消費者**（seam の背後）。seed された合成マスター（`master.summary_json`）から決定的に取得して生成する。[[マスター元データ]]を実際に作る[[突合]]は slice-11/12（Phase 3）の仕事で、slice-08 は待たない。→ **依存は seed のみ**（確定済み報告＝slice-03 は全体の前提だが、slice-08 が直接読むのは seed のマスター）。
2. **ファイル名の `YYYYMMDD`（出力日）はパターンで固定。** テストは命名規則（`^[スタッフ名]_.+_\d{8}\.xlsx$`）を検証し、日付の実値には依存しない。サーバは注入可能なクロックで日付を付与。
3. **レスポンスは メタデータ＋構造化内容。** AC-2（数値創作なし）を API 層で観測できるよう、`POST /skill-sheets` の 201 は メタデータ に加え **AI言い換えの固定スキーマ JSON（アンカー対応のフィールド値）** を返す。
4. **slice-08 は生成側のみ。** 再生成は**新 id/file_url（別ストレージオブジェクト）の非破壊作成**。表示 filename は format に一意子が無いため同日再生成で重複しうる（一意性は id/URL 側）。「旧が残る／履歴一覧」の観測は slice-09（S6）へ。
5. **`POST /skill-sheets` は任意 `staff_id`（省略＝自分）。** 他人の `staff_id` を対象にした要求は 403（deny-by-default・backend 強制）、未認証は 401。管理者の代行生成は Phase 4。

## AC-1 生成は3フェーズ分離で、データ取得と書き込みは決定的に行う

- source: reference-mock        # spec.md §3.5
- **Given** seed された合成[[マスター元データ]]（`master.summary_json`）を持つスタッフ
- **When** `POST /skill-sheets` で生成を実行する
- **Then** 「データ組立（seed のマスターから決定的に取得）→ AI言い換え（固定 JSON スキーマ・言い換えのみ）→ テンプレート反映」の順で処理され、**AI言い換えの構造化結果がレスポンスに含まれる**
- **テスト境界**: AI言い換えの構造化出力（固定スキーマ）はレスポンスで観測する。**「テンプレート反映（アンカー位置・書式維持）」の xlsx 内部検証は API 非観測のため slice-08 では行わず、slice-09（プレビュー/DL）・slice-10（アンカー検証）へ委ねる**（黙って省略しない・明示記録）。

## AC-2 マスターに無い数値を AI が創作しない

- source: reference-mock        # spec.md §3.5・§5（数値・事実を創作させない）
- **Given** 定量値を含まない[[マスター元データ]] fixture
- **When** `POST /skill-sheets` で AI言い換えを行う
- **Then** レスポンスの構造化内容に**数字が現れない**。かつ**中身は空でない**（空振り合格を防ぐ・件数ガード）。出力はアンカー対応の固定 JSON スキーマに縛られる（slice-02 AC-3 と同型の検証）

## AC-3 出力ファイル名をサーバ側で機械的に付与する

- source: reference-mock        # spec.md §3.5
- **Given** 生成の実行日（サーバの注入可能クロック）
- **When** スキルシートを生成する
- **Then** ファイル名は `[スタッフ名]_[ファイル名]_YYYYMMDD.xlsx` 形式でサーバ側が機械的に付与する。テストは**命名パターン**（`^テスト 太郎_.+_\d{8}\.xlsx$`）を固定し、日付の実値には依存しない

## AC-4 生成物は（合成）ストレージ格納＋署名付き URL。再生成は新ファイルを作る（非破壊）

- source: reference-mock        # spec.md §3.5（格納と履歴）／メモリモードでは合成
- **Given** すでに1度スキルシートを生成済みのスタッフ
- **When** 再生成する
- **Then** 201 のメタデータに**合成の署名付き URL**（非実在・非空の URL 文字列）が含まれる。再生成は**新しい id と file_url（別ストレージオブジェクト）**を返し、旧生成物を上書きしない（新規作成）。表示 filename は同日再生成で重複しうる（一意性は id/URL 側）
- **スコープ境界**: 「旧生成物が履歴一覧に残る」ことの観測は **slice-09（S6 生成履歴）**が担う。slice-08 は「再生成で別物の新ファイルができる（非破壊）」までを固定する

## AC-5 生成 API の REST 表現と認可

- source: PM        # ★参照モックは HTTP 表現を規定しない（overview §3）
- 理由: 振る舞いは answer key 由来だが、パス・メソッド・ステータス・認可境界は新規設計。
- **Given** 認証済みスタッフ
- **When** `POST /skill-sheets`（本文に任意 `staff_id`。省略＝自分）を呼ぶ
- **Then**
  - 自分（または省略）を対象 → `201` と生成シートのメタデータ＋構造化内容
  - 他人の `staff_id` を対象 → `403`（deny-by-default・backend で強制）
  - 未認証 → `401`
