# slice-20 wellbeing-soft-questions — ソフト設問・ウェルビーイング

> 振る舞いの正本: `reference-mock/spec.md §3.7`（ソフト設問・ウェルビーイング）・`§6.2`（プライバシー）・
> `reference-mock/phase2-design.md §4`（雑感の閲覧範囲）。
> 依存: slice-19（役割タグ付き設問セットの上に載る）。

## AC-1 ソフト設問が役割で流れ先を分かれる

- source: reference-mock        # spec.md §3.7 表（AI活用→スキル抽出／課題・所感→内部非反映／雑感→ウェルビーイング除外）
- **Given** 「AIを業務に活かせたこと」「課題」「所感」「雑感（メンタル面）」を含む設問セット
- **When** 報告を要約・スキルシート生成する
- **Then**「AI活用」はスキルとしてシートへ反映され、「課題」「所感」は内部の振り返りとしてシート非反映、「雑感」はウェルビーイングとして後段へ渡さない

## AC-2 雑感は AI 変換・シート反映から完全に除外される

- source: reference-mock        # spec.md §3.7・§6.2・overview §5「雑感は AI 変換の対象から完全に除外」
- **Given** 雑感（メンタル面）を含む報告
- **When** 要約・専門用語化・スキルシート生成のいずれの AI 処理を実行する
- **Then** 雑感の本文は `Summarizer` 抽象化層へ一切渡らず、AI 出力にもシートにも一切現れない

## AC-3 雑感の閲覧が限定される

- source: reference-mock        # phase2-design §4「閲覧は本人／主担当管理者／メンタルケア担当のみ。副担当・一般管理者・super admin は閲覧不可」
- 補足: 認可失敗の HTTP コードは 🟥 PM（`403`・deny-by-default・overview §3）
- **Given** 雑感を含む報告と、閲覧権限を持たない利用者（副担当・担当外の管理者）
- **When** その利用者が雑感を参照しようとする
- **Then** 雑感は返らず `403` 相当で遮断される。本人・限定閲覧者にのみ表示される
- 未定: 主担当／副担当／メンタルケア担当の役割モデルの実装解決順は Phase 2 の権限3軸に依存（overview §8 項目9〜14・phase2-design §8）。雑感の**任意入力・本人による非公開**の詳細は未定（spec.md §9 論点4・overview §8 項目4「雑感の閲覧範囲・任意/非公開の詳細」）。

## AC-4 AI は状態の診断・点数化をしない

- source: reference-mock        # spec.md §3.7「AIは状態の診断・点数化を行わない。スコア化・監視ダッシュボードにしない」・overview §6 非ゴール
- **Given** 雑感を含む報告
- **When** システムが雑感を処理する
- **Then** メンタル状態のスコア・診断・点数は一切生成されず、監視ダッシュボード等のスコア化表示も存在しない

## 画面要件

- 対象画面: 報告入力の設問部（S3 業務報告入力の拡張。ソフト設問は slice-19 の設問セットとして S10 で構成し、回答は報告入力で行う）
- golden 撮影: 入力フォームは可／雑感の非表示は DOM アサーションで検証（共有ビューに雑感が出ないこと）
- **UI-AC（source: PM）**
  - 理由: 参照モックは画面の DOM・文言を規定しない。ソフト設問の見え方と雑感の秘匿はここで確定する。
  - ソフト設問（雑感／所感／課題／AI活用）が報告入力の設問部に表示され回答できる。
  - 雑感は権限を持たない利用者の報告閲覧ビューに**表示されない**（DOM 上に出ない）。
  - 雑感をスコア・点数として提示する UI が存在しない。

## 合成フィクスチャ（PM 所有）

**雑感は L2（要配慮個人情報）＝最も厳格に扱う。合成データのみ。実データ・本番ダンプは一切入れない**（憲法 §1-6・7）。
