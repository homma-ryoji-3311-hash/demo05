# slice-23 ai-follow-up-question — AI 追加質問

> 振る舞いの正本: `reference-mock/report-quality-design.md`（AI 追加質問の全設計。特に §2 フロー・§3 粒度判定・
> §5 回答の扱い・§6 必須/任意・§10 degrade・§12 スライス順）。**【決定】のみ AC 化し、【未決】は「未定」と明記する。**
> REST 表現は参照モックに無い＝設計（`overview.md §3` `POST /reports/:id/follow-up`・確定は slice-03 の `POST /reports/:id/confirm`）。
> 依存: slice-03（要約の確認・確定）・slice-22（グループ別設定の設定駆動基盤）。画面: S4 AI要約 確認・編集 の拡張。

## AC-1 要約後に薄い項目へ一度だけ追加質問する

- source: reference-mock        # report-quality §2.1「要約を作った後に薄い項目へ追加質問」・§2.2「1報告につき1ラウンド」・§3.1「対象カテゴリ×ルール検出（決定的）が主役」
- **Given** 要約済みの下書き報告で、管理者が選んだ対象カテゴリに属する項目がルール検出（空・極端に短い等）で薄いと判定される
- **When** 追加質問を要求する（`POST /reports/:id/follow-up`）
- **Then** 薄い対象カテゴリについて追加質問が**一度だけ**生成・提示される。同じ入力なら同じ判定になる（ルール検出は決定的）。Gemini 自動判定は薄さ判定の主役にしない

## AC-2 回答を本文へ追記し要約を作り直す

- source: reference-mock        # report-quality §5.1「回答は raw_text へ追記（確定前のみ）」・§5.2「本文追記後に要約を作り直す」
- **Given** 追加質問への回答（確定前の下書き段階）
- **When** 回答を送信する
- **Then** 回答が報告本文（`raw_text`）へ追記され、要約（`ai_summary_json`）が作り直される。報告は下書きのままで、確定済み報告の本文・要約は変えない（原則6）

## AC-3 必須未回答は確定ブロック・任意はスキップ可

- source: reference-mock        # report-quality §6.1・§6.2「必須未回答は確定ブロック／任意はスキップして確定可」
- 補足: 確定ブロック時の HTTP コードは 🟥 PM（`422` 相当・未回答は検証失敗扱い・CLAUDE.md §6）
- **Given** 提示された追加質問のうち、**必須**の質問が未回答
- **When** 報告を確定しようとする（`POST /reports/:id/confirm`）
- **Then** 確定はブロックされる（答えるまで確定できない）。一方、**任意**の質問は未回答でもスキップして確定できる

## AC-4 質問自体が出せなかった degrade 時は必須ブロックを発動しない

- source: reference-mock        # report-quality §6.2「必須ブロックは『提示されたのに未回答』時に限る」・§10.1「機能が落ちても報告は必ず出せる」・§10.3
- **Given** AI 失敗等で追加質問**自体が生成・提示されなかった**（degrade）状態
- **When** 報告を確定しようとする
- **Then** 必須ブロックは発動せず、追加質問を省いて通常どおり確定できる（「問われていない」状態を確定不能にしない）

## AC-5 対話は一度きりで二重質問しない

- source: reference-mock        # report-quality §2.2・§9.3「質問済み状態で二重質問を防ぐ」・§10.6「再試行しても二重質問にしない」
- **Given** すでに追加質問ラウンドを済ませた報告
- **When** 再度追加質問を要求する、または再試行が走る
- **Then** 新しい質問ラウンドは走らない（質問済み状態により一度きりが保証される）

## 画面要件

- 対象画面: S4 AI要約 確認・編集 の拡張（追加質問を確認・確定フロー内に差し込む）
- golden 撮影: 追加質問の提示〜回答〜再要約の基本フローは可／**再来訪 UI は項目20 が未定のため golden 対象外**（DOM アサーションで基本挙動を検証）
- **UI-AC（source: PM）**
  - 理由: 参照モックは画面の DOM・文言を規定しない。追加質問の提示・必須制御・degrade 表示はここで確定する。
  - 薄い項目への追加質問が確認画面に提示され、その場で回答を入力できる。
  - **必須**の質問が未回答のとき「確定」操作が無効化される（テキストで理由を示す・色のみに頼らない）。**任意**はスキップして確定できる。
  - degrade 時（質問が出せなかった）は追加質問を表示せず、通常どおり確定できる。

## 合成フィクスチャ（PM 所有）

**本番/実データは一切入れない**（憲法 §1-6）。回答は本文へ追記されるため、雑感等 L2 を対象カテゴリに含めない（雑感は AI 変換対象外＝slice-20）。

| フィールド | 階層 | 例 | 備考 |
|---|---|---|---|
| report.status | L0 | `draft`（確定前のみ質問） | 確定後は不変 |
| target_categories | L0 | `["issues"]`（単一基準の仮値） | 対象カテゴリ設定（項目16 は未定） |
| summary.issues | L0 | `[""]`（空＝ルール検出で薄い） | 薄い項目のトリガ |
| follow_up.required | L0 | `true` / `false` | 必須/任意（AC-3） |
| follow_up.answer | L0 | `"○○の手順で△△を設定し××を確認"` | 本文へ追記 |
| follow_up.state | L0 | `asked` / `degraded` | 一度きり・degrade 切り分け（AC-4・AC-5） |
