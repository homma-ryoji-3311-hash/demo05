# 必要画面・機能一覧

`docs/spec.md` をもとに、業務報告・スキルシート生成システムで必要となる画面と機能を実装・画面設計向けに整理する。

## 1. 利用者と権限

| 利用者       | 主な目的                                                                         | 権限範囲                     |
| ------------ | -------------------------------------------------------------------------------- | ---------------------------- |
| スタッフ     | 業務報告の入力、AI整形結果の確認、自身のスキルシート閲覧・ダウンロード、通知設定 | 自分の報告・スキルシートのみ |
| 営業・管理者 | 担当スタッフの状況確認、設問・Excelテンプレート管理、スキルシート生成・一括出力  | 担当グループのスタッフのみ   |
| HR / 担当者  | 雑感・メンタル面のケア対応                                                       | 本人と限定担当者のみ         |

権限境界は画面表示だけでなく、必ずバックエンド側で強制する。

## 3. 主要機能

### 認証・権限

- Google OAuth 2.0でログインする。
- 法人GWSアカウント、許可ドメイン、招待制で社外ユーザー混入を防ぐ。
- ユーザーはロールと所属グループを持つ。
- スタッフは自分のデータのみ、管理者は担当グループのみアクセスできる。

### 業務報告入力

- 入力方式は自由文、設問回答、音声入力を提供する。
- 音声はSTTでテキスト化し、最終的に編集可能な本文へ集約する。
- 前回の本文・回答・AI要約を読み取り専用で参照できる。
- 入力内容は常時自動保存し、通信断や離席でも失わない。

### AI要約・確認

- AI処理は提供元非依存の抽象化層の背後で実行する。
- 生入力を対応案件、インシデント、成果、課題、抽出スキルなどへ構造化する。
- AI結果は確定前に必ず人が確認・編集できる。
- 元入力と要約を並べて照合できる。
- 不足や不確実な箇所は要確認フラグとして表示する。
- AIは本文にない数値・事実を創作しない。

### 報告確定・突合

- 確定した報告は生報告ログとして不変保存する。
- 生報告ログと突合済みマスター元データは分離する。
- 確定のたびに、既存マスターと新報告を案件キー単位で増分マージする。
- インシデントなどのステータスは単純追記ではなく最新状態へ更新する。

### スキルシート生成

- マスター元データ、案件、スキル集計をDBから決定的に取得する。
- AIは職務経歴書向けの言い換えのみを担い、事実や数値は創作しない。
- AI出力はアンカー対応JSONスキーマに固定する。
- openpyxlでExcelテンプレートのアンカー位置へ書き込む。
- 生成ファイルはオブジェクトストレージに保存し、履歴と署名付きURLをDBで管理する。
- プレビューはPDFまたはHTML、ダウンロードはxlsxで提供する。

### 設問テンプレート管理

- 管理者はグループ単位で設問セットを作成・編集・並べ替えできる。
- 設問ごとに回答形式、必須/任意、役割タグを設定する。
- 役割タグで案件キー、ステータス、スキル抽出、内部のみなどの流れ先を決める。
- 公開前に必須ロールの不足を検証する。
- 報告は作成時のテンプレート版を保持し、過去報告を壊さない。

### 通知

- スタッフごとにリマインド時刻を設定できる。
- Slack通知を主経路とし、メール通知はオプションとする。
- 短間隔ジョブで通知対象ユーザーをDBから抽出して送信する。
- タイムゾーンは保存UTC、表示・判定はユーザーのローカル時刻に統一する。

### 管理者コンソール

- 担当グループのスタッフ一覧を表示する。
- 氏名、客先、最終報告、報告状況、最新スキルシート、操作を一覧化する。
- 未報告者、要確認残件、未生成スキルシートを確認できる。
- 個別プレビュー、個別ダウンロード、個別再生成、全員分生成、一括ZIP出力を提供する。

## 5. 共通非機能要件

- PC/スマートフォン両対応。スタッフ画面はモバイルファースト、管理者画面はPCでの一覧性を重視する。
- 状態は色だけでなくテキストでも表現する。
- キーボード操作だけで主要操作を完結できる。
- 通信断、録音権限拒否、STT失敗、AI失敗時も入力内容を失わない。
- 生成物・テンプレート・音声は安全に保存し、配布は署名付きURLを使う。
- 客先機密情報を扱う前提で、権限・ログ・保存先を設計する。

## 6. 未決・要確認事項

- スタッフの複数グループ所属を許可するか。
- 管理者の複数グループ兼任を許可するか。
- グループ移管時の過去報告・マスターデータの扱い。
- 雑感の閲覧範囲、任意/非公開の詳細仕様。
- STT方式の選定。
- AI要約モデルの選定。
- 突合処理を同期実行のままにするか、規模拡大時に夜間バッチへ移行するか。
- スキルシートプレビュー方式をPDFにするかHTMLにするか。
